# 功能说明文档

本文档说明专业考勤系统各功能的内部逻辑，包括前端入口、API 调用链、后端服务流转、数据库读写和数据流向。

## 1. 用户认证与会话

### 功能入口

- 页面：`frontend/login.html`
- 前端请求封装：`frontend/src/utils/request.js`
- 后端控制器：`backend/app/controllers/auth_controller.py`
- 路由注册：`backend/app/__init__.py`

### API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| POST | `/api/auth/login` | 登录并返回 access token 和 refresh token |
| POST | `/api/auth/refresh` | 刷新 token |
| GET | `/api/auth/me` | 获取当前登录用户 |

### 内部逻辑

1. 用户在 `login.html` 输入用户名和密码。
2. 前端调用 `/api/auth/login`。
3. `LoginResource.post()` 使用 `login_parser` 解析请求。
4. 后端查询 `user_info` 表，使用 bcrypt 校验密码。
5. 登录成功后生成 JWT，并写入 `session_version`。
6. 前端将 token 保存到 `sessionStorage`。
7. 之后所有 API 请求由 `request.js` 自动附加 `Authorization: Bearer <token>`。
8. 后端 JWT 中间件校验 token、过期时间和 `AUTH_SESSION_VERSION`。
9. 后端服务重启后，新的 `AUTH_SESSION_VERSION` 会使旧 token 失效。

### 数据流向

```text
login.html
  -> POST /api/auth/login
  -> auth_controller.LoginResource
  -> user_info
  -> JWT token
  -> sessionStorage
```

## 2. 实时考勤

### 功能入口

- 页面：`frontend/src/views/AttendancePage.vue`
- 摄像头组件：`frontend/src/components/Attendance/AttendanceCamera.vue`
- 前端 API：`frontend/src/api/attendance.js`
- 后端控制器：`backend/app/controllers/attendance_controller.py`
- 业务服务：`backend/app/services/attendance_service.py`
- 识别服务：`backend/app/services/face_recognition_service.py`

### API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| POST | `/api/attendance/checkin` | 上传拍照结果并完成考勤 |
| GET | `/api/attendance/records` | 查询考勤记录 |

### 考勤完整链路

1. `AttendanceCamera.vue` 调用浏览器摄像头。
2. 用户点击拍照后，前端将视频帧绘制到 canvas，并导出 base64 图片。
3. `AttendancePage.vue` 调用 `checkin()`，提交 `image_base64`、`image_format` 等数据。
4. `CheckinResource.post()` 解析请求并获取当前用户。
5. `AttendanceService.checkin()` 首先做幂等检查。
6. 活体检测：
   - 单帧走 `LivenessDetector.detect_liveness_from_base64()`。
   - 多帧走 `detect_liveness_multi_frame_base64()`。
7. 活体失败：
   - 写入失败考勤记录。
   - 返回照片攻击、视频重放或活体失败错误码。
8. 活体通过后进入人脸识别：
   - 学生账号登录时只与本人特征比对。
   - 教师/管理员场景可与全体学生特征比对。
9. `FaceRecognitionService.recognize_single_face()` 解码图片。
10. `FaceFeatureExtractor.detect_face()` 检测人脸并生成对齐后的人脸张量。
11. `FaceFeatureExtractor.extract_feature_from_detection()` 提取 512 维特征。
12. `FaceMatcher.find_best_match()` 与学生库特征计算余弦相似度。
13. `EmotionClassifier.classify_emotion()` 对人脸区域进行情绪识别。
14. 保存考勤图片到上传目录。
15. 写入 `attendance_record` 表。
16. 如有情绪结果，写入 `emotion_record` 表。
17. 返回签到结果、学生信息、匹配置信度、活体分数和情绪。

### 数据流向

```text
摄像头画面
  -> base64 图片
  -> /api/attendance/checkin
  -> 活体检测
  -> 人脸检测与对齐
  -> FaceNet 特征提取
  -> 学生库特征比对
  -> 情绪识别
  -> attendance_record / emotion_record
  -> 前端考勤结果
```

## 3. 合照识别

### 功能入口

- 页面：`frontend/src/views/GroupPhotoPage.vue`
- 上传组件：`frontend/src/components/GroupPhoto/PhotoUpload.vue`
- 前端 API：`frontend/src/api/groupPhoto.js`
- 后端控制器：`backend/app/controllers/group_photo_controller.py`
- 业务服务：`backend/app/services/report_service.py`
- 多人脸检测：`backend/app/services/multi_face_detector.py`

### API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| POST | `/api/group-photo/recognize` | 上传合照并识别学生 |
| GET | `/api/group-photo/records` | 查询合照识别记录 |

### 合照识别完整链路

1. 用户在 `PhotoUpload.vue` 上传 JPG/PNG 合照。
2. 前端将文件转为 base64。
3. `groupPhoto.js` 调用 `/api/group-photo/recognize`。
4. 后端兼容 `photo_base64` 和 `image_base64` 字段。
5. `ReportService.process_group_photo()` 调用 `FaceRecognitionService.recognize_multiple_faces()`。
6. `MultiFaceDetector.detect_faces()` 检测合照中所有人脸：
   - 优先使用 `facenet-pytorch MTCNN keep_all=True`。
   - 生成每张人脸的检测框、置信度、对齐张量。
   - 不可用时降级为 OpenCV Haar。
7. 对每张人脸执行：
   - 对齐张量特征提取。
   - 与学生库比对。
   - 情绪识别。
8. 保存合照原图。
9. 写入 `group_photo_record` 表：
   - 活动名称、活动时间、照片名称、识别人数等。
10. 写入 `group_photo_recognition_detail` 表：
   - 每张人脸对应的学生、置信度、情绪、识别状态。
11. 写入 `emotion_record` 表。
12. 返回识别到的学生列表。

### 数据流向

```text
合照文件
  -> base64
  -> /api/group-photo/recognize
  -> 多人脸检测
  -> 每张脸特征提取和比对
  -> group_photo_record
  -> group_photo_recognition_detail
  -> emotion_record
  -> 前端识别结果
```

## 4. 学生管理

### 功能入口

- 页面：`frontend/src/views/StudentPage.vue`
- 前端 API：`frontend/src/api/student.js`
- 后端控制器：`backend/app/controllers/student_controller.py`
- 业务服务：`backend/app/services/student_service.py`
- 数据访问：`backend/app/repositories/student_repository.py`

### API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | `/api/students` | 分页查询学生 |
| POST | `/api/students` | 新增学生 |
| GET | `/api/students/<student_id>` | 查询学生详情 |
| PUT | `/api/students/<student_id>` | 更新学生信息 |
| DELETE | `/api/students/<student_id>` | 删除学生 |
| POST | `/api/students/batch-import` | JSON 批量导入学生 |
| POST | `/api/students/batch-import/excel` | Excel 批量导入学生 |

### 新增学生链路

1. 教师或管理员在学生管理页面填写学生信息并上传人脸图片。
2. 前端调用 `POST /api/students`。
3. `StudentListResource.post()` 校验角色权限。
4. `StudentService.create_student()` 检查学号是否重复。
5. 调用 `FaceFeatureExtractor.extract_feature_from_base64()` 提取人脸特征。
6. 写入 `student_info` 表。
7. 创建对应 `user_info` 账号。
8. 返回学生基础信息。

### 批量导入链路

JSON 批量导入：

```text
前端 JSON students 数组
  -> /api/students/batch-import
  -> StudentService.batch_import()
  -> 循环 create_student()
  -> 返回成功/失败统计
```

Excel 批量导入：

```text
Excel 文件
  -> pandas.read_excel()
  -> 校验 student_id/name/class_name
  -> 转为 students 数组
  -> StudentService.batch_import()
```

## 5. 情绪分析

### 功能入口

- 页面：`frontend/src/views/AnalysisPage.vue`
- 图表组件：`frontend/src/components/Emotion`
- 前端 API：`frontend/src/api/emotion.js`
- 后端控制器：`backend/app/controllers/emotion_controller.py`
- 数据访问：`backend/app/repositories/attendance_record_repository.py`

### API

| 方法 | 路径 | 功能 |
| --- | --- | --- |
| GET | `/api/emotion/statistics` | 情绪分布统计 |
| GET | `/api/emotion/trend` | 情绪趋势统计 |

### 内部逻辑

1. 考勤或合照识别时产生情绪结果。
2. 后端将情绪、置信度、来源类型、学生、专业、检测时间写入 `emotion_record`。
3. 前端分析页请求统计接口。
4. 后端按专业、学生、来源、时间范围过滤。
5. `find_emotion_statistics()` 按情绪类型聚合数量。
6. `find_emotion_trend()` 按日期聚合情绪分布。
7. 前端使用 Chart.js 展示饼图、柱状图或趋势图。

## 6. 报表导出

### 功能入口

- 组件：`frontend/src/components/Report/ReportExport.vue`
- 前端 API：`frontend/src/api/report.js`
- 后端控制器：`backend/app/controllers/report_controller.py`
- 业务服务：`backend/app/services/report_service.py`

### API

| 方法 | 路径 | 报表 |
| --- | --- | --- |
| GET | `/api/reports/attendance/export` | 考勤明细报表 |
| GET | `/api/reports/attendance-summary/export` | 考勤汇总报表 |
| GET | `/api/reports/activity-frequency/export` | 活动频次报表 |
| GET | `/api/reports/activity/export` | 学生活动参与报表 |
| GET | `/api/reports/activity-record/export` | 活动记录报表 |

### 考勤汇总报表

输入：

- 专业，可选。
- 开始时间，必填。
- 结束时间，必填。

内部逻辑：

1. 查询指定时间范围内 `status=1` 的考勤记录。
2. 按学生去重，得到已签到学生集合。
3. 查询指定专业下所有状态正常的学生。
4. 将学生拆分为已签到和未签到。
5. 使用 Pandas + OpenPyXL 生成 Excel。
6. 输出两个 Sheet：`已签到学生`、`未签到学生`。

### 学生活动参与报表

输入：

- 学生学号，必填。

内部逻辑：

1. 查询 `group_photo_recognition_detail` 中该学生识别成功的记录。
2. 关联 `group_photo_record` 获取活动名称、活动时间和照片名称。
3. 导出活动名称、活动时间、照片名称、匹配置信度、情绪、识别时间。
4. 输出 Sheet：`活动记录`。

### 活动记录报表

输入：

- 活动名称，必填。

内部逻辑：

1. 查询指定活动名称下所有合照记录。
2. 关联识别详情表，筛选识别成功的学生。
3. 同一学生重复出现时，只保留匹配置信度最高的一条。
4. 导出活动名称、活动时间、学号、姓名、专业、照片名称、匹配置信度、情绪、识别时间。
5. 输出 Sheet：`签到学生名单`。

### 数据流向

```text
ReportExport.vue
  -> report.js
  -> /api/reports/*/export
  -> report_controller
  -> report_service
  -> repositories
  -> pandas/openpyxl
  -> xlsx blob
  -> 浏览器下载
```

## 7. 权限控制

### 角色

| 角色 | 权限 |
| --- | --- |
| admin | 管理员，拥有全部管理、查询、导出权限 |
| teacher | 教师，可管理学生、查看记录、导出报表 |
| student | 学生，只能查看和操作与本人相关的数据 |

### 权限实现

- JWT 校验由 `flask-jwt-extended` 处理。
- 当前用户通过 `get_current_user()` 从 token 读取。
- 学生数据访问通过 `enforce_student_data_access()` 限制。
- 学生不能查看其他学生的详情或考勤。
- 报表导出和学生管理仅教师/管理员可用。

## 8. 统一响应与错误处理

### 成功响应

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "request_id": "req_xxx"
}
```

### 错误响应

```json
{
  "code": 40001,
  "message": "字段不能为空",
  "data": null,
  "request_id": "req_xxx"
}
```

前端 `request.js` 会解析后端返回的 `message`、`msg` 或 `error` 字段，并在页面中显示明确错误原因。

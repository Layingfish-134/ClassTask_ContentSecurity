# 项目架构文档

本文档说明专业考勤系统的目录结构、模块职责、分层设计和主要文件作用。

## 1. 总体架构

项目采用前后端分离架构：

```text
浏览器前端
  -> Vue 页面与组件
  -> Axios 请求 /api
  -> Flask REST API
  -> Service 业务层
  -> Repository 数据访问层
  -> SQLAlchemy ORM
  -> SQLite 数据库
```

算法能力作为后端服务层的一部分存在，由考勤、合照、学生管理和报表功能复用。

```text
图像输入
  -> OpenCV 解码
  -> MTCNN 人脸检测和对齐
  -> FaceNet 特征提取
  -> 余弦相似度匹配
  -> 情绪识别
  -> 业务记录入库
```

### 4.1 应用工厂

文件：`backend/app/__init__.py`

职责：

- 创建 Flask 应用。
- 加载 `.env`。
- 配置 JWT、CORS、上传大小、数据库。
- 初始化数据库。
- 注册统一异常处理。
- 注册 REST API 路由。
- 初始化默认账号。
- 生成本次服务启动的 `AUTH_SESSION_VERSION`，用于重启后使旧 token 失效。

### 4.2 config

```text
backend/app/config/
├── database_config.py
└── file_storage_config.py
```

| 文件 | 作用 |
| --- | --- |
| `database_config.py` | 创建 SQLAlchemy 实例，配置 SQLite，初始化表结构 |
| `file_storage_config.py` | 管理上传目录和上传路径 |

### 4.3 controllers

```text
backend/app/controllers/
├── attendance_controller.py
├── auth_controller.py
├── emotion_controller.py
├── group_photo_controller.py
├── report_controller.py
└── student_controller.py
```

控制器层负责：

- 接收 HTTP 请求。
- 使用 parser 解析参数。
- 调用 Service 层。
- 处理权限和响应格式。

| 文件 | 主要职责 |
| --- | --- |
| `auth_controller.py` | 登录、刷新 token、当前用户、默认账号初始化 |
| `attendance_controller.py` | 考勤签到、考勤记录查询 |
| `student_controller.py` | 学生增删改查、批量导入 |
| `group_photo_controller.py` | 合照识别、合照记录查询 |
| `emotion_controller.py` | 情绪统计、情绪趋势 |
| `report_controller.py` | 各类 Excel 报表导出、文件访问 |

## 10. 配置说明

后端主要配置来自 `backend/.env`：

| 配置 | 说明 |
| --- | --- |
| `SECRET_KEY` | Flask 密钥 |
| `JWT_SECRET_KEY` | JWT 签名密钥 |
| `UPLOAD_FOLDER` | 上传目录 |
| `MAX_CONTENT_LENGTH` | 最大上传大小 |
| `FACE_MATCH_THRESHOLD` | 人脸匹配阈值 |
| `FACE_CROP_MARGIN` | 人脸裁剪边距 |
| `FACE_MIN_SIZE` | 最小人脸尺寸 |
| `LIVENESS_THRESHOLD` | 活体检测阈值 |

前端主要配置在：

```text
frontend/src/config/index.js
```

包括 API 基础路径、请求超时、文件大小限制、摄像头默认分辨率等。

## 11. Git 清理建议

以下内容不建议提交：

- `__pycache__/`
- `*.pyc`
- `venv/`
- `node_modules/`
- `frontend/dist/`
- `backend/instance/`
- `backend/uploads/`
- `backend/models/`
- `*.db`
- `*.pt`
- `*.bin`

这些规则已经写入 `.gitignore`。如果历史上已被 Git 跟踪，需要使用 `git rm --cached` 或删除后提交清理。
